# Android 应用结构与优化审查

- 审查日期：2026-02-07
- 项目：`areyoulu-main`
- 范围：React + Vite + Capacitor(Android) 的结构、构建配置、隐私安全、性能与可维护性

## 总结

当前应用功能完整且可用（打卡、日历、统计、PIN 锁、CSV 导入导出、分享）。  
主要风险集中在隐私安全、离线稳定性与 Android 发布配置。

## 建议修复顺序

1. 先做隐私与安全加固：
- 移除前端 API Key 注入。**🔴 已完成**
- 将明文 PIN/安全答案替换为更安全的存储方案，并移除直接展示 PIN 的流程。**🔴 已完成**
- 复核备份策略（`allowBackup` / data extraction rules）。

2. 再做离线与依赖治理：
- 将运行时 CDN/字体/图标/背景依赖替换为本地打包资源。

3. 再做 Android 发布就绪：
- 去掉硬编码 `org.gradle.java.home`。
- 调整导出存储路径策略以适配新 Android。
- 启用并验证 release 混淆压缩策略。

4. 最后做性能与可维护性：
- 拆分 `App.tsx` 的状态与业务逻辑到模块或 hooks。
- 加强 CSV 导入健壮性与去重。
- 将组件内联样式迁移到静态 CSS。

### 本轮优化结果（2026-02-18）

已完成：
- PIN 防爆破：增加失败次数限制与 30 秒临时锁定，并持久化锁定状态（重启后仍生效）（`components/LockScreen.tsx`、`constants.ts`）。
- 本地数据清理收敛：`localStorage.clear()` 改为仅清理应用自身 key（`App.tsx`）。
- 导出路径优化：改为 `Directory.Documents/lulemo`，并保留 Cache 兜底（`App.tsx`）。
- Android 隐私/发布配置：`allowBackup=false`、移除旧外部存储权限、开启 release 混淆（`android/app/src/main/AndroidManifest.xml`、`android/app/build.gradle`）。
- CSV 健壮性：导出增加标准转义；导入改为更可靠解析；导入时按 `id` 去重（`App.tsx`、`components/SettingsView.tsx`）。
- Toast 稳定性：修复多次触发时定时器覆盖问题（`App.tsx`）。
- 工程治理：补充 `LICENSE`，删除弃用占位文件（`LICENSE`、`components/DeerParticleSystem.tsx`）。
- 品牌命名修正：用户可见与工程展示命名统一为 `lulemo`（`README.md`、`metadata.json`、`package.json`、`App.tsx`、`LICENSE`）。
- 存储键迁移：应用存储键由 `lulemo_*` 升级为 `lulemo_*`，并在启动时自动迁移旧键，确保历史数据不丢（`constants.ts`、`App.tsx`）。
- 图标离线化（单项）：移除 Font Awesome CDN，改为本地图标子集（`@fortawesome/react-fontawesome` + 按需图标），减少运行时第三方请求（`index.html`、`components/FaIcon.tsx`、各页面图标调用）。
